# ADR-001: Core Tech Stack Selection for JobPilot

## Context

JobPilot requires a solid foundation to handle complex business logic (AI extraction, real-time Kanban, cronjobs) while ensuring rapid development speed for an MVP. The system handles mixed data types: highly structured (application pipeline) and unstructured (HTML from job boards).

## Decision

We decided to adopt the following stack:

1. **TypeScript (Monorepo with pnpm & Turborepo):** Ensures end-to-end type safety from Frontend to Backend, reducing runtime errors. `pnpm` ensures strict dependency resolution without phantom dependencies, while `Turborepo` enables highly optimized, cached parallel builds.
2. **Backend - NestJS:** Provides a clear modular architecture (Dependency Injection) suitable for Clean Architecture, with built-in support for Queues (BullMQ) and Cronjobs.
3. **Frontend - React (Vite):** Rich ecosystem, combined with TanStack Query for state management and optimistic updates on the Kanban board.
4. **Database - Polyglot Persistence:**
   - **PostgreSQL:** Acts as the "Source of Truth" for core entities. Utilizes the `pgvector` extension to calculate vector distance (match score between CV and JD).
   - **MongoDB:** Manages raw JDs and flexible extracted data (schema-less), preventing the relational database from bloating.

## Consequences (Trade-offs)

- **Pros:** Independent scalability of components, easy separation of API and Workers later. Optimal tools for specific data structures.
- **Cons:** Higher operational/DevOps overhead to maintain two databases (PostgreSQL & MongoDB) in parallel. This will be mitigated by using Docker Compose for the local development environment.
